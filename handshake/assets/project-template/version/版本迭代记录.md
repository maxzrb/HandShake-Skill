# 版本迭代记录

面向读者：项目用户、项目负责人、未来接手者。

用途：用容易阅读的中文记录项目版本变化。这个文件只用于人工阅读，不作为 Codex 的项目管理依据。Codex 的管理依据仍然是 `AGENTS.md` 和 `docs/codex/`。

## 版本规则

- 推荐使用语义化版本：`MAJOR.MINOR.PATCH`。
- 破坏性变更升级主版本。
- 兼容性新增升级次版本。
- 修复、说明优化、非行为变更升级补丁版本。
- 已发布版本不应静默改写，后续修改应发布新版本。
